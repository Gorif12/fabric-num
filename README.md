# 基于 Hyperledger Fabric 的房地产交易系统

## 系统架构设计

### 1. 组织(Organization)设计

#### Org1: 房地产管理机构（房管局）

- 负责房产信息的认证
- 负责产权变更的审核
- 管理房产相关证件

#### Org2: 金融机构（银行）

- 负责资金托管
- 处理交易支付

### 2. 用户设计

系统用户包括：

- 买家：查看房产信息，发起购买请求，支付房款
- 卖家：发布房产信息，确认交易
- 房管局管理员：验证房产信息，办理产权变更
- 银行管理员：处理资金托管和转账

### 3. 用户交互流程

```
用户（买家/卖家）-> 应用程序界面 -> API服务 -> 组织节点 -> 区块链网络
```

### 4. 系统实现

#### 4.1 应用层设计

- 用户管理系统（使用传统数据库）
- 用户注册、登录等基础功能
- 不同类型用户的权限管理

#### 4.2 区块链层设计

- 房产信息上链
- 产权转移的智能合约
- 资金托管和转账的智能合约

### 5. 业务流程示例

房产交易流程：

1. 卖家在系统中挂牌房产
    - 通过 Org1（房管局）验证房产信息
    - 将房产信息上链
2. 买家提出购买请求
    - 通过 Org2（银行）进行资金托管
3. 交易执行
    - Org1 验证产权信息
    - Org2 处理资金转移
    - 智能合约自动执行产权转移

### 6. 权限管理

#### Org1 权限

- 房产信息认证
- 产权变更
- 证件管理

#### Org2 权限

- 资金托管操作
- 交易确认
